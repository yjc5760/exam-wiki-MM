## 1. 知識庫架構一覽

知識庫分為**七個知識層**，前三層由 Cowork 透過 compile/ingest 生成，後四層由 Cowork 直接維護：

| 層 | 目錄 | 內容 | 生成方式 |
|----|------|------|---------|
| Layer 1 | `wiki/concepts/` | 核心概念（形心/慣性矩/主應力/撓度/挫屈/塑性等 32 個） | Cowork (compile-all) |
| Layer 1 | `wiki/problems/` | 題目解析頁 | Cowork (ingest) |
| Layer 2 | `wiki/philosophy/` | 解析哲學（彈性分析/塑性分析/疊加/相容性/能量法） | Cowork (compile-all) |
| Layer 3 | `wiki/methods/` | 解題方法論（Mohr's circle/積分法/Euler/機構法） | Cowork (compile-all) |
| Layer 4 | `wiki/diagnosis/` | 題型診斷決策樹（8 頁） | Cowork (直接存入) |
| Layer 5 | `wiki/failure-modes/` | 失效模式（降伏/挫屈/斷裂/過大變形） | Cowork (直接存入) |
| Layer 6 | `wiki/materials/` | 材料行為（彈塑性鋼材/脆性材料/材料常數） | Cowork (直接存入) |
| Layer 7 | `wiki/code-ref/` | 核心公式彙整（截面性質/應力/變位/挫屈/塑性） | Cowork (直接存入) |

**主導航入口：** `wiki/index.md`（七層全覽）、`wiki/by-year.md`（依考年）

## 2. 核心工作流程

```
【更新驗算狀態後 ingest（所有題目已解析）】
  你驗算某題後對 Cowork 說：「將 MM-XXXX-N 的 verificationStatus 改為 verified」
    ↓
  對 Cowork 說：「ingest MM-XXXX-N」→ wiki 自動更新

【補充截圖】
  你截圖（chart/eqn/hand）存入 raw/solutions/MM-XXXX-N/ 資料夾
    ↓
  對 Cowork 說：「MM-XXXX-N 補充截圖完成，請更新索引」

【新增概念/方法】
  對 Cowork 說：「add concept [概念名]」或「add method [方法名]」
    ↓ 引導建立 concepts.json 條目 + wiki 頁面

【複習查詢】
  直接向 Cowork 提問，或瀏覽 wiki/ 目錄
```

> **所有操作都對 Cowork 說即可，不需要終端機。**

## 3. Cowork 指令速查

所有指令分三大類，完整說明見 [CLAUDE-CODE.md](CLAUDE-CODE.md)。

### 3.1 raw 資料維護

| 觸發語句 | 功能 |
|---------|------|
| `將 MM-XXXX-N 的 verificationStatus 改為 verified` | 人工驗算通過後標記 |
| `將 MM-XXXX-N 的 verificationStatus 改為 needs-review` | 發現錯誤時標記 |
| `MM-XXXX-N 補充截圖完成，請更新索引` | 同步 hasHandwritten 等欄位 |

### 3.2 wiki 編譯與維護（7個）

| 觸發語句 | 功能 | 前置條件 |
|---------|------|---------|
| `ingest MM-XXXX-N` | 將一道題目寫入 wiki | verificationStatus = "verified" |
| `compile all` | 從 raw/ 重建整個 wiki（concepts/methods/philosophy/traps） | — |
| `lint wiki` | 健檢 wiki 完整性（16 項） | — |
| `status` | 驗證進度統計 | — |
| `reindex` | 修正 hasSolution 不一致 | — |
| `add concept [名稱]` | 新增概念到 concepts.json | — |
| `add method [名稱]` | 新增解題方法論頁面 | — |

### 3.3 備考分析類（5個）

| 觸發語句 | 功能 |
|---------|------|
| `frequency` | 統計各考點歷年出現次數，找出高頻考點 |
| `analyze YYYY` | 深度分析某一年考卷的考點分布 |
| `predict` | 根據歷年頻率推測最可能出現的考點 |
| `study MM-UN` | 彙整某單元所有題目、重點公式、常見陷阱（輸出 study-MM-UN.html） |
| `study MM-UN-n` | 深度複習某命題子項（七區塊格式：命題分析/截面圖解/解題流程/公式/考題清單/陷阱/互動計算） |
| `refresh-dashboard` | 重新生成離線儀表板（dashboard.html + dashboard-data.js） |

### 3.4 查詢快捷類（4個）

| 觸發語句 | 功能 |
|---------|------|
| `find [關鍵字]` | 搜尋含某關鍵字的題目（如 `find Mohr's circle`） |
| `related MM-XXXX-N` | 找出與某題考點相似的其他題目 |
| `unverified` | 列出所有有解析但尚未驗算的題目 |
| 直接提問 | 自由查詢知識庫 |

## 4. 複習使用指南

### 4.1 依考年複習

開啟 `wiki/by-year.md` → 選年份 → 點入題目頁 → 查看核心考點與解題步驟。

### 4.2 依題型複習（診斷決策樹）

| 題型 | 診斷頁 |
|------|--------|
| 梁截面應力（彎曲+剪力） | `wiki/diagnosis/beam-stress.md` |
| 扭力桿件 | `wiki/diagnosis/torsion.md` |
| 應力轉換 / Mohr's circle | `wiki/diagnosis/stress-transformation.md` |
| 梁撓度與斜率 | `wiki/diagnosis/beam-deflection.md` |
| 柱挫屈臨界載重 | `wiki/diagnosis/buckling.md` |
| 塑性分析（極限載重） | `wiki/diagnosis/plastic-analysis.md` |
| 軸力桿件（含不靜定/熱應力） | `wiki/diagnosis/axial-member.md` |
| 殘留應力 | `wiki/diagnosis/residual-stress.md` |

### 4.3 查詢核心公式（code-ref/）

開啟 `wiki/code-ref/index.md` 查詢截面性質、應力、撓度、挫屈、塑性公式。

### 4.4 術語速查

開啟 `wiki/concepts/` 查詢個別概念說明。

### 4.5 直接問 Cowork

```
形心怎麼計算？
T 形截面的最大剪應力在哪裡？
Mohr's circle 的 τ 方向怎麼定？
MM-2016-1 的考點是什麼？
哪幾題考過 Euler 挫屈？
塑性斷面模數 Z 和彈性斷面模數 S 的差別？
```

## 5. 常見問題

**Q：解析 .md 放在哪裡？**
- 原始來源（唯讀）：`raw/solutions/MM-XXXX-N/MM-XXXX-N.md`
- 知識庫版本（ingest 後）：`wiki/problems/MM-XXXX-N.md`

**Q：如何更正已 verified 的答案？**
對 Cowork 說：「將 MM-XXXX-N 的 verificationStatus 改為 needs-review」→ 說明錯誤 → 重新驗算 → 改回 verified → 說「ingest MM-XXXX-N」。

**Q：新增的概念頁 / 方法頁需要 ingest 嗎？**
不需要。`wiki/concepts/`、`wiki/methods/`、`wiki/code-ref/` 等由 Cowork 直接維護，不走 ingest。

**Q：截圖要怎麼命名？**

| 類型碼 | 內容 | 範例 |
|--------|------|------|
| `fig` | 題目附圖 | `MM-2016-1-fig-1.png` |
| `chart` | 設計圖表 | `MM-2016-1-chart-1.png` |
| `eqn` | 公式截圖 | `MM-2016-1-eqn-1.png` |
| `hand` | 手寫補充 | `MM-2016-1-hand-1.png` |

**Q：2005 年和 2009 年有 5 題，是否正確？**
是。這兩年各有 5 道題目，question_index.json 已包含所有 96 題（2002–2025 年）。

**Q：目前解析進度？**
對 Cowork 說 `status` 即可取得最新統計。目前概況：
- 解析資料夾：96 個（2002–2025 年全部）
- hasSolution=true：96 題
- verified：0 題（待人工驗算）
